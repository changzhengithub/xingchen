# arco-design的使用与问题

arco-design是字节跳动新开发的前端UI模板组件，使用和说明都很少，vue版是在vue3.0版上开发的，所以使用它必须使用vue3来进行开发。

安装

# npm
npm install --save-dev @arco-design/web-vue
# yarn
yarn add --dev @arco-design/web-vue

引入

import { createApp } from 'vue'
import App from './App.vue';
import ArcoVue from '@arco-design/web-vue';
import '@arco-design/web-vue/dist/arco.css';

const app = createApp(App);
app.use(ArcoVue);
app.mount('#app');

# createApp(App).use(ArcoVue).mount('#app');

# upload组件使用

upload组件给的示例很简单，简单说明一下使用情况。

首先不论使用它的默认上传还是自定义请求上传都是直接请求接口自动上传，选一个上传一个。然后通过各种事件来给前端用来显示和定义上传的过程，如进度条、加载中、上传完成、失败提示等，我们不自己定义就会显示它默认的样式，文件列表提示等。

它给的自定义上传请求示例和使用action一样，就是自定义了请求的方式，通过option参数在请求中返回各种状态，定义好之后，还是一样在组件上通过error/success这几种方法让前端显示上传过程，本质还是和action一样的。上传只能一个一个上传，或者multiple多选直接上传。不过单个的替换头像这种操作还是直接可以使用的。

所以如果我们想要那种不直接上传，而是把上传的文件显示出来，等我们一起调用接口再上传该如何使用呢？ 我们可以借用自定义上传行为 custom-request 来实现，通过custom-request的参数获取文件内容来自定义显示文件列表，然后还可以通过on-before-upload来预处理文件，过滤提示，再然后调接口统一上传，根据接口的状态显示上传的加装进度、过程、提示等。

或者使用:auto-upload="false"不让组件自动上传只选择文件，然后获取到文件列表再自己调用上传接口。

<template>
    <div class="picture-list" v-for="(item, index) in urlList" :key="index">
        <img :src="item">
    </div>
    <a-upload 
        v-if="urlList.length < 5"
        list-type="picture-card"
        :show-file-list="false"
        :on-before-upload="beforeUpload"
        :custom-request="customRequest"
    >
    </a-upload>
</template>

<script setup>
import { ref } from 'vue'

const fileList = ref([])
const urlList = ref([])

const beforeUpload = (file) => {
    console.log(file);
    const { size, type } = file
    const limitType = type == 'image/png' || type == 'image/jpeg'
    const limitSize = size < 1024 * 1024 * 10
    if (!limitType) {
        alert('请上传png或者jpg格式图片!');
    }
    if (!limitSize) {
        alert('文件不可大于 10MB!');
    }
    return limitSize && limitType
}

const customRequest = (option) => {
    const { fileItem } = option
    const { file, url } = fileItem;
    fileList.value.push(file)
    urlList.value.push(url)
}

</script>
