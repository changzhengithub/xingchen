# 搭建Eslint编码规范项目

使用 vue-cli 创建3.0版本项目，并搭建 standard 风格编码规范。

# 安装

升级vue-cli，最新版本就不需要升级了。

npm install -g @vue/cli

# 创建

执行命令

vue create vue-test

执行创建命名后，选择 Manually select features 选项，不要选前两个默认选项，然后手动选择 Babel、Router、Vuex、CSS Pre-processors、Linter / Formatter等选项，然后选择版本2.x或者3.x，然后输入Y回车，然后选择less或sass，然后选择编码规范 ESLint + Standard config 为我们所常用的编码规范。然后接着回车就创建好了。

# 修改编码规范

添加好 standard 编码规范后，有些规范是我们不需要的，我们可以根据Eslint的配置方法在 .eslintrc.js文件的 rules 规则中进行配置，去除不必要的规范。如果哪些检查不合适，可以再去Eslint官网去查找添加到rules中关闭。

module.exports = {
  root: true, // 从根文件开始
  env: {
    node: true
  },
  extends: [
    'plugin:vue/vue3-essential',
    '@vue/standard' // standard风格
  ],
  parserOptions: {
    parser: '@babel/eslint-parser'
  },
  // eslint配置参数
  rules: {
    'vue/multi-word-component-names': ['off'], // 是否强制组件名称多单词
    'camelcase': 'off', // 是否强制使用驼峰拼写法命名规定
    'space-before-function-paren': 'off', // 是否强制方法圆括号左边空格
    'no-multiple-empty-lines': 'off', // 最后一行保留一行空格
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off'
  }
}

配置好后还需要去除强制被检查的文件夹，我们可以添加 .eslintignore 文件进行去除。

/node_modules/
/public/
/src/assets/
/src/store/
/src/utils/
.vscode
.idea

# 格式化文件

使用vscode打开项目时，可以使用Ctrl+Shift+P打开配置使用Beautify来格式化JS、CSS、HTML，但是对vue项目单文件并不有好，还有一个Vetur插件也可以用来格式化，这是打开vue项目必装的插件，右键选择格式化文档就可以对vue文件进行格式化，但是格式化的风格不符合Eslint + Standard 编码规范，我们可以通过在setting.json对Vetur格式化进行配置来达到所期望的效果。

打开 文件-首选项-设置在最右上角有个图标打开设置json，点击打开setting.json，在里面就只可以直接编辑想要的风格。配置vetur的默认选项为prettier就可以说实现对应的格式化，不需要下载Prettier插件。如果不想在本地保存配置，可以在项目中添加 .prettierrc 文件进行配置，一般会先识别项目自带的格式化文件，然后才会使用本地的。

// setting.json
{
  "editor.fontSize": 18,
  // tab 大小为2个空格
  "editor.tabSize": 2,
  // 保存时格式化
  // "editor.formatOnSave": true,
  // 开启 vscode 文件路径导航
  // "breadcrumbs.enabled": true,

  // 默认使用prettier格式化支持的文件
  "vetur.format.defaultFormatterOptions": {
    // prettier里的配置
    "prettier": {
      // 结尾无分号
      "semi": false,
      // 超过200个字符换行，配置html属性过多自动换行
      "printWidth": 200,
      // 使用单引号
      "singleQuote": true,
      // 无尾随逗号
      "trailingComma": "none",
      // 箭头函数单个参数不加分号
      "arrowParens": "avoid"
    },
  },
}

// .prettierrc
{
  "semi": false,
  "printWidth": 200,
  "singleQuote": true,
  "trailingComma": "none",
}

# 问题

1、less-loader版本过高问题

安装预编译插件less时 less-loader 版本过高导致在vue.config.js中配置 less 参数 javascriptEnabled 编译失败。一种做法是降低版本，但是vue-cli版本过高降不成功，二是更改less配置。

const path = require('path')
const { defineConfig } = require('@vue/cli-service')
module.exports = defineConfig({

  // 正常
  css: {
    loaderOptions: {
      less: {
        modifyVars: {
          // less vars，customize ant design theme
          // 'primary-color': '#F5222D',
        },
        // DO NOT REMOVE THIS LINE
        javascriptEnabled: true,
      }
    }
  },

  // 更改后
  css: {
    loaderOptions: {
      less: {
        lessOptions: {
          // DO NOT REMOVE THIS LINE
          javascriptEnabled: true,
          globalVars: {
            // less vars，customize ant design theme
            // 'primary-color': '#F5222D',
          }
        }
      }
    }
  },
})
